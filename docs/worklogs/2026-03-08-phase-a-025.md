# 2026-03-08 Phase A 025 - Add pilot label and superrare placeholder configuration

Added the minimum contract/deploy configurability needed for an optional self-only mainnet pilot that stays clearly non-official while preserving mint logic.

## What changed
1. Updated `foxar/src/CoreCats.sol` so collection name and symbol are deploy-time constructor inputs instead of hard-coded labels.
2. Updated `foxar/src/CoreCatsMetadataRenderer.sol` so token-name prefix, token description, and optional superrare placeholder mode are deploy-time inputs.
3. Added neutral programmatic placeholder rendering for the two logo-bearing superrare visuals when pilot placeholder mode is enabled.
4. Updated `foxar/script/CoreCatsDeploy.s.sol` to accept pilot-oriented env overrides while keeping the official `CoreCats` / `CCAT` path as the default.
5. Updated `foxar/script/CoreCatsPostDeployCheck.s.sol` so pilot deploys can also assert expected collection name and symbol.
6. Added contract tests covering custom labels and superrare placeholder mode.
7. Updated `docs/MAINNET_PILOT_FALLBACK.md` with the exact env knobs for a non-official pilot deploy.

## Result
1. The repo can now deploy either the official labeled contract or a clearly non-official pilot from the same code path.
2. Pilot mode can avoid displaying the two logo-bearing superrare visuals without changing mint/security/randomness/supply behavior.
3. The official release path still requires a separate final canary on the real CoreCats contract.
