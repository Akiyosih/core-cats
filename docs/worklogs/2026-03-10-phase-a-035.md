# 2026-03-10 Phase A 035 - Record the successful CCATTEST pilot QR entry path

Recorded the missing public evidence about how the successful self-only `CCATTEST` mainnet pilot mint was actually entered, so the repository distinguishes proven paths from still-unverified ones.

## Pilot mint route that is now explicitly recorded
1. The successful mint was a closed self-only pilot against the public origin `https://core-cats.vercel.app`.
2. At the time of that mint, the closed pilot was still pinned to one known wallet via `COREPASS_EXPECTED_CORE_ID`.
3. The successful entry path did not use the public `/mint` UI.
4. The successful entry path used a Codex-generated CorePass QR request outside the public `/mint` UI.
5. The QR was scanned with the device's standard camera and then handed off to CorePass.
6. This success record does not count as evidence for the CorePass in-app QR scanner path.

## End-to-end evidence from that pilot mint
1. `identify` succeeded for the pilot wallet and the session advanced to the signed commit request.
2. `commitMint(...)` succeeded on the pilot contract:
   - tx hash: `0x20cbe8cb29d489edeff166e4e447216bfd014c7a66314060a1ef4f4399e973c6`
   - block: `16739661`
3. relayer `finalizeMint(minter)` succeeded:
   - tx hash: `0x9e10146973db5aec0820e32816fc40b963edea84cb4e43eb4d14a9f5ee4e0460`
   - block: `16739643`
4. The assigned token was `CCATTEST #996`.
5. `tokenURI(996)` was decoded successfully to on-chain JSON/SVG for the pilot token.

## What this does and does not prove
1. It proves that one real mainnet CorePass route succeeded end-to-end through `Vercel -> Contabo backend -> CorePass -> CCATTEST pilot contract`.
2. It does not prove the public `/mint` UI route yet.
3. It does not prove the CorePass in-app QR scanner route yet.
4. It does not prove generic-wallet behavior yet, because the pilot was still self-only at that point.

## Launch consequence
1. The repository should treat this as valid pilot evidence, not as proof that every intended public entry path is ready.
2. Release readiness still requires separate canaries for:
   - public `/mint`
   - device standard camera -> CorePass launch under the generic-wallet path
   - CorePass in-app QR scanner
   - non-self-only wallets
