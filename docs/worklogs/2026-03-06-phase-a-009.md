# Worklog: Phase A-009 (Permissionless Finalize Upgrade)

Phase: A
Date: 2026-03-06
Result: success

## Why this step was needed
The earlier `commit-reveal` branch was transparent, but it still allowed selective non-reveal by the minter. That was not strong enough for the project's "open, trustworthy, and auditable" standard.

## What changed
1. Replaced user-driven `revealMint(secret)` with permissionless `finalizeMint(minter)`.
2. Kept the same broad randomness family:
   - future block boundary
   - public chain data
   - lazy Fisher-Yates assignment
3. Removed the requirement that the minter disclose a secret at finalize time.
4. Derived assignment entropy from:
   - stored `commitHash`
   - `blockhash(finalizeBlock)`
   - `minter`
   - `address(this)`
   - `block.chainid`
   - `finalizeBlock`

## Why this is better
1. A minter can no longer refuse to reveal only because they dislike the result.
2. Any third party can finalize an eligible pending commit.
3. The fairness story is easier to explain publicly from on-chain data alone.
