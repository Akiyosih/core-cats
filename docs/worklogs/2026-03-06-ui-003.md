# 2026-03-06 UI 003 - Shift mint UX to CorePass QR/app-link

## Summary
Reworked the active `/mint` implementation so the public wallet flow no longer depends on an injected browser provider.

## Why
CorePass official documentation centers QR/app-link protocol requests rather than a standard injected desktop provider flow.
For this project, that is the correct chain-native UX to build around first.

## What changed
1. Added a CorePass mint session layer in the Next.js server runtime.
2. Added CorePass protocol request generation for:
   - `corepass:sign`
   - `corepass:tx` for `commitMint(...)`
   - `corepass:tx` for manual `finalizeMint(minter)` fallback
3. Added callback routes for both QR callback and app-link return paths.
4. Replaced the active `/mint` page workflow so it now drives:
   - identify via CorePass sign
   - commit via CorePass tx
   - relayer auto-finalize when available
   - CorePass finalize fallback when relayer is unavailable
5. Updated README/spec/transparency notes to match the new CorePass-first mint path.

## Important limitation
The current CorePass mint session store is in-memory only. This is acceptable for local/testnet work but should be replaced with durable shared storage before public production release.
