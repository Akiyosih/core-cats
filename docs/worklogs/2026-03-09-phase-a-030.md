# 2026-03-09 Phase A 030 - Prune operator-specific security detail from public docs

Reduced public operational detail so the repository keeps verifiable trust boundaries and deploy flow, while role assignments, funding state, and secret-handling choreography remain in private ops notes.

## What changed
1. Rewrote public pilot/backend docs to prefer role-based wording such as:
   - deployer
   - signer
   - finalizer
2. Removed unnecessary wallet-number references from public worklogs where they were only describing operator-side setup history.
3. Trimmed public backend guidance so it no longer prescribes exact secret filenames or host-side courier choreography.

## Public vs private boundary
1. Public docs should explain:
   - contract behavior
   - deploy/verify sequence
   - off-chain trust boundaries
   - required runtime interfaces
2. Private ops notes should retain:
   - wallet numbering
   - funding state
   - host-specific paths
   - secret handoff timing
   - live runtime snapshots

## Result
1. The public repository still explains what outside readers need in order to evaluate the system.
2. The public repository no longer mirrors operator-specific security posture that does not materially improve third-party verification.
