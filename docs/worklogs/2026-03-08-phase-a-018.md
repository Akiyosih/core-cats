# 2026-03-08 Phase A 018 - Prepare Vercel mainnet cutover and dynamic transparency state

Prepared the public-site side for the mainnet launch sequence before the final operator deploy window.

## What changed
1. `web/lib/server/core-env.js` now prefers `CORE_RPC_URL` and also accepts `CORE_EXPLORER_BASE_URL` on the server side.
2. `web/app/transparency/page.js` now reads the configured network, launch state, and contract address instead of hard-coding the Devin rehearsal contract link.
3. Added `web/.env.production.example` for the intended Vercel production env shape.
4. Added `docs/VERCEL_MAINNET_CUTOVER_CHECKLIST.md` with the operator steps for `closed -> canary -> public`.
5. Updated the web README and architecture doc to point at the new production env and cutover artifacts.

## Result
1. The public site can remain live on mainnet before the final contract address exists.
2. The transparency page can now show `Mainnet deployment pending` instead of leaking the old Devin rehearsal link.
3. The repo now contains the missing Vercel-side env template and cutover checklist.
