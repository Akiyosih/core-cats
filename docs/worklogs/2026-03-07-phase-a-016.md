# Phase A-016

## Summary

Refined the split mint backend design so that:

1. the Vercel app keeps the CorePass session flow, QR/app-link generation, and callback redirect UX
2. the Contabo backend owns durable session storage, mint authorization issuance, and finalize relayer execution

This avoids reimplementing Core-specific calldata/session behavior in a second runtime while still removing signing keys and in-memory persistence from Vercel.

## Changes

1. added `mint-backend/` Python service scaffold
2. added `MINT_BACKEND_API.md`
3. added `CONTABO_MINT_BACKEND_RUNBOOK.md`
4. updated Vercel-side server code to support:
   - external session persistence
   - external mint authorization
   - external finalize relay
5. updated architecture docs to match the refined split
