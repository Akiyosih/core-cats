## Summary

## What was added
1. Server-side mint authorization route:
   - `/api/mint/authorize`
2. Server-side relayer finalize route:
   - `/api/mint/finalize`
3. New Foxar script for authorization signing:
   - `foxar/script/CoreCatsIssueMintAuthorization.s.sol`
4. Client mint workflow:
   - commit with wallet
   - try relayer-assisted auto-finalize
   - expose manual finalize fallback

## Design notes
1. Core signing stays on the server side through `spark` instead of re-implementing Core signature handling in JavaScript.
2. The contract remains permissionless.
3. Relayer finalize is convenience, not a trust dependency.
