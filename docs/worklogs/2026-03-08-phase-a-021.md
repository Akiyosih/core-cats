# 2026-03-08 Phase A 021 - Add public freeze/renounce and minter-privacy notes

Added explicit public-facing disclosure for two trust-sensitive areas that were previously only implicit in the repo and launch notes.

## What changed
1. Added `docs/FREEZE_AND_RENOUNCE_POLICY.md` to explain:
   - deployer is temporary
   - ownership should move to the cold owner wallet after setup
   - owner actions during launch/live mint are exceptional
   - renounce is a later hard-freeze option, not an automatic promise at first deploy
2. Added `docs/MINTER_PRIVACY_NOTE.md` to explain:
   - the current mint path is Protocol-direct, not Connector/KYC-transfer
   - extra KYC is not intended for the current release
   - on-chain activity and backend/session coordination still exist
   - the mint should not be marketed as anonymous
3. Updated `docs/TRUST_AND_PRIVACY_SURFACE.md`, `README.md`, and `/transparency` so those new notes are reachable from the public review surface.

## Result
1. The public repository now explains both:
   - the current owner/admin freeze posture
   - the actual privacy expectations for minters
2. Public readers no longer need to infer these points from scattered implementation details alone.
