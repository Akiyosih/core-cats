# 2026-03-08 Phase A 019 - Allow Foxar keystore handling for Wallet 2 and Wallet 4

Reduced unnecessary raw-key dependence for the current launch path by aligning deploy/finalize handling with Foxar's official keystore wallet options.

## What changed
1. Updated the deploy and set-signer scripts so they can broadcast without `DEPLOYER_PRIVATE_KEY` when `spark script` is invoked with a keystore.
2. Updated the finalize script so it can run in keystore mode for Wallet 4 while still keeping the old raw-key path available.
3. Extended the Python mint backend config so production may use:
   - raw signer key for Wallet 3
   - raw finalizer key, or
   - finalizer keystore path + password file for Wallet 4
4. Updated env examples, runbook text, and tests to match the new finalizer keystore option.

## Result
1. Wallet 2 can now stay as `UTC` if deploy/setup commands use Foxar keystore flags.
2. Wallet 4 can now stay as `UTC` if the Contabo backend is configured with the keystore path and password file.
3. Wallet 3 still remains the raw-key requirement under the current mint-authorization design.
