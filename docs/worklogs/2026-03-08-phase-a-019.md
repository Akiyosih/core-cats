# 2026-03-08 Phase A 019 - Allow Foxar keystore handling for deploy and finalize roles

Reduced unnecessary raw-key dependence for the current launch path by aligning deploy/finalize handling with Foxar's official keystore wallet options.

## What changed
1. Updated the deploy and set-signer scripts so they can broadcast without `DEPLOYER_PRIVATE_KEY` when `spark script` is invoked with a keystore.
2. Updated the finalize script so it can run in keystore mode for the finalizer role while still keeping the old raw-key path available.
3. Extended the Python mint backend config so production may use:
   - raw signer key for the signer role
   - raw finalizer key, or
   - finalizer keystore path + password file for the finalizer role
4. Updated env examples, runbook text, and tests to match the new finalizer keystore option.

## Result
1. The deploy/admin role can now stay on the official keystore path if deploy/setup commands use Foxar keystore flags.
2. The finalizer role can now stay on the official keystore path if the backend is configured with the keystore path and password file.
3. The signer role still remains the raw-key requirement under the current mint-authorization design.
