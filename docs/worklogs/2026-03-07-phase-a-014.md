# 2026-03-07 Phase A 014 - Record optional self-only mainnet pilot fallback

## Summary
Documented the optional fallback path to use only if CorePass testnet validation is still unavailable and the project does not want the first operator mint to live on the official CoreCats contract.

## What changed
1. Added `docs/MAINNET_PILOT_FALLBACK.md`.
2. Updated `docs/WORK_PROCEDURE_CORE_BLOCKCHAIN.md` so the default path remains the official contract, while the self-only pilot remains an explicit optional fallback.
3. Updated `docs/MAINNET_CLOSED_LAUNCH_RUNBOOK.md` so the pilot path is described as an insertable preflight step rather than a replacement for the official canary.

## Key decision
1. Default path remains:
   - official contract
   - `closed -> canary -> public`
2. Optional fallback path is:
   - self-only mainnet pilot
   - official final canary
   - public launch
3. Pilot success does not remove the need for an official canary because the signing domain still changes with the contract address.

## Next action
Continue web/launch-state and production-ops preparation. If the project later chooses the pilot fallback, add pilot label configurability without changing mint/security logic.
