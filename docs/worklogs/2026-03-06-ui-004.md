# 2026-03-06 UI 004 - Record CorePass local validation limit

## Summary
Documented the current practical limit for live CorePass mint validation in the active local environment.

## What was confirmed
1. The available CorePass app exposes only a mainnet `cb...` account.
2. It does not currently expose a Devin testnet `ab...` account.
3. Because the Core Devin rehearsal target requires testnet transaction capability, live CorePass E2E on Devin is still blocked at the wallet layer.

## Why this matters
Without this note, the repository could falsely imply that the new CorePass QR/app-link `/mint` flow has already been fully exercised on Devin mobile/desktop. That would be inaccurate.

## Resulting interpretation
1. The CorePass-first `/mint` implementation remains the intended production-target UX.
2. Core Devin contract/randomness/relayer validation still stands.
3. CorePass desktop/mobile E2E should be treated as pending until a testnet-capable CorePass path is available.
